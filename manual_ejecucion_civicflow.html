1. Prerrequisitos de Software

No necesitas descargar ni configurar el framework Astro de manera global en tu máquina. El proyecto se encarga de autodescargarlo localmente utilizando Node.js.

  • Descargar e instalar Node.js: Visita nodejs.org y descarga la versión recomendada LTS (versión 22 o superior). Al instalarlo, se añadirá el gestor de paquetes npm a tu sistema.
  • Editor de Código: Se recomienda Visual Studio Code para editar y correr la terminal.
  • Git (Opcional): Para clonar el repositorio de forma ágil desde la consola.

2. Configuración del Proyecto

Sigue esta secuencia de pasos para descargar e instalar todas las dependencias necesarias:

  1. Abre Visual Studio Code.
  2. Ve a Archivo > Abrir Carpeta y selecciona la carpeta frontend (es la carpeta que contiene el archivo package.json).
  3. Abre la terminal integrada de VS Code (atajo: Ctrl + ` o en el menú superior Terminal > Nueva terminal).
  4. Ejecuta el siguiente comando para instalar las librerías necesarias: 
    npm.cmd install

    (Si utilizas macOS o Linux, simplemente ejecuta npm install).

💡 Nota sobre dependencias: Este comando crea la carpeta node_modules localmente, la cual aloja el compilador de Astro y Tailwind CSS v4 de forma temporal. No se sube al repositorio de Git.

3. Cómo Correr el Servidor Local

Una vez completada la instalación, ejecuta el siguiente comando para correr el servidor de desarrollo local:

npm.cmd run dev

(En macOS o Linux, corre npm run dev).

Esto habilitará la dirección web local: http://localhost:4321/. Ábrela en tu navegador para ver la maquetación en vivo.

Compilación del Catálogo Completo

Si realizas cambios en los archivos y necesitas volver a generar el catálogo de pantallas offline de la raíz del proyecto (CivicFlow_Catalogo_Pantallas.html), ejecuta estos dos comandos en orden:

npm.cmd run build
node compile_screens.js

4. Subir el Repositorio a GitHub

El proyecto ya incluye un archivo .gitignore configurado para evitar subir archivos residuales pesados. Puedes compartir este repositorio en GitHub sin problemas:

  1. Inicializa Git en la carpeta raíz del proyecto: 
    git init
  2. Añade los archivos al historial: 
    git add .
  3. Crea un commit para guardar el estado actual: 
    git commit -m "feat: maqueta civicflow finalizada"
  4. Vincúlalo con tu nuevo repositorio remoto y súbelo a GitHub: 
    git branch -M main
git remote add origin https://github.com/TU_USUARIO/TU_REPOSITORIO.git
git push -u origin main

5. Resolución de Errores Comunes de PowerShell

Si al correr comandos con npm obtienes un error del sistema que indica que la ejecución de scripts está deshabilitada:

  • Añade la extensión .cmd a tus comandos, por ejemplo, utiliza npm.cmd install y npm.cmd run dev.
  • O bien, en tu terminal de VS Code, presiona el menú desplegable junto al icono de basura y selecciona Command Prompt (Símbolo del sistema). Esto evitará las políticas restrictivas de PowerShell.
  • O desbloquea temporalmente PowerShell ejecutando: 
    Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
